# N08 Template Update Design

## Goal

让 `document-filler` 使用 `C:\Users\p\Desktop\新分支20260721` 提供的新模板生成三份受影响材料，同时保持台账兼容、工单聚合、数据目录匹配、流程图和目录更新逻辑不变。

## Scope

本次只修改以下三类材料的生成行为：

- `N08-数据报表服务 / 02-数据报表_设计文档`
- `N08-数据统计分析 / 01-数据统计分析_需求文档`
- `N08-数据统计分析 / 02-数据统计分析_设计文档`

模板文件不纳入仓库：仓库当前只维护生成逻辑、规则和测试，桌面目录中的模板作为本次生成输入。

## Design

### 1. 数据报表设计文档

继续使用现有目录代码和数据目录字段匹配。对每个数据来源目录：

- 四级标题使用数据目录的中文资源名；
- 数据来源表的合并标题使用“资源英文编码 + 空格 + 资源中文名”；
- 保留字段明细、数据内容截图和数据处理逻辑；
- 不再追加“报表指标设计”标题和指标表。

没有中文名或英文编码时，分别回退到已有目录代码，避免生成空标题；其他目录匹配行为不变。

### 2. 数据统计分析需求文档

需求来源段落只写“服务周期内，共有 N 张需求单，M 张工单涉及 K 次数据统计分析”，不追加旧的“如下表”套话。`2.3` 下的两个模板小节作为独立锚点：

- `2.3.1 需求单、工单产出对应列表`：填充原五列表格，按工单聚合并保留合计行；
- `2.3.2 工单与任务的对应关系`：按结果表程序逐行填充四列表格，序号自增，任务中文名取结果表清单的中文部分。

每个工单的“数据加工要求”保留原三条数据来源字段，只把第三条正文标签从“数据量对后续运维的特殊要求”改为“对运维的工作要求”。

表格定位使用章节标题后的第一个表格，不依赖固定 `doc.tables[index]`，从而兼容修订记录或模板增加表格的情况。

### 3. 数据统计分析设计文档

需求来源表按每个结果表一行生成四列：序号、需求单编号、工单编号、程序名称。程序名称使用结果表中文名，移除工单名称列。

顶层 `Heading 1` 仍以新模板的“数据统计分析设计”作为内容截断锚点；每个程序内部的 `Heading 3` 从“数据统计分析设计”改为“数据结果表设计”。实体表、结果表字段、流程图和数据加工逻辑的来源与顺序保持不变。

## Error Handling

- 缺少必要章节或目标表格时抛出带章节名称的 `ValueError`，不生成伪造占位材料。
- 结果表没有中文名时回退英文名；目录资源缺少名称时回退目录编码。
- 旧台账列名继续通过现有规范化别名读取。

## Verification

- 为三个行为分别增加回归测试，并先确认测试在旧代码上失败；
- 全量 `python -m unittest discover -s tests`；
- 对三份真实输出检查章节标题、表头、行数、旧章节缺失和无 `待补充/未匹配到` 占位；
- 用 Word/Office 更新目录并导出 PDF，检查无表格溢出、空白页和标题/表格错位。
