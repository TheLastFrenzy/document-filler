# 验收文档自动填充技能 (Document Filler)

## 用途

根据《台账清单》(.xlsx) 中指定服务目录的筛选数据，填充指定材料类型对应的 Word 验收文档模板或 Excel 说明材料模板。
新版台账列名以 `工单标题`、`程序数`、`工单描述`、`结果表清单`、`源表对应目录代码清单`、`附件` 和 `执行周期` 为准；脚本仍兼容旧列名 `工单内容`、`报表统计次数`、`业务说明/业务描述`、`统计分析结果表清单`、`02-数据报表_设计文档-数据来源库表清单对应数据目录代码`、`交付物`、`自测报告附件`、`03-数据统计分析_测试文档_工单自测报告附件` 和 `数据统计分析执行周期`。新旧列同时存在时优先读取新列。

支持的**材料类型**：
- `01-API接口开发_需求文档` — API 接口需求文档（业务场景 + 需求清单 + 工单需求口径 + 从自测报告解析接口输入/输出参数）
- `02-API接口开发_数据模型设计` — API 数据模型设计文档（保留模板静态章节 + 按工单和接口生成 4.2 接口出入参配置表）
- `03-API接口开发_接口开发代码` — API 接口开发代码文档（接口开发列表 + 按接口抽取命名规范性和代码检查截图）
- `04-API接口开发_接口测试报告` — API 接口测试报告（API 测试清单 + 按接口生成输入/输出参数表和测试结果截图）
- `05-API接口开发_作业上线记录` — API 作业上线记录（作业上线清单 + 按接口抽取共享接口命名规范性截图）
- `01-库表落地方式_需求文档` — 库表落地方式需求文档（需求来源表格 + 按工单生成库表落地明细，明细来自 `附件` 内嵌 Excel）
- `02-库表落地方式_数据模型设计` — 库表落地方式数据模型设计文档（业务逻辑说明 + 目标表数据量截图 + 库表说明 + 推送频率）
- `04-库表落地方式_测试报告` — 库表落地方式测试报告（库表落地测试清单 + 上线交付截图 + 测试结论表）
- `05-库表落地方式_作业上线记录` — 库表落地方式作业上线记录（库表落地上线清单，明细来自台账和 `附件` 内嵌 Excel）
- `06-库表落地方式_共享记录` — 库表落地方式共享记录（数据库表落地验证清单 + 源表/目标表数据量截图验证记录）
- `01-数据报表_需求文档` — 需求文档（需求来源表格 + 需求内容 + 内嵌报表清单）
- `01-数据统计分析_需求文档` — 数据统计分析需求文档（需求来源表格 + 需求内容 + 统计分析结果表清单 + 业务逻辑说明）
- `02-数据报表_设计文档` — 设计文档（业务分析表格 + 数据内容截图 + 数据来源库表清单 + 数据处理逻辑；来源标题使用资源中文名，来源表首行为“资源英文编码 资源中文名”，不生成报表指标设计章节）
- `02-数据统计分析_设计文档` — 数据统计分析设计文档（需求来源程序列表 + 实体表清单 + 结果表结构 + 数据处理流程图 + 数据加工逻辑）
- `03-数据报表_上线记录` — 上线记录（需求来源表格 + 产出说明 + 上线交付截图 + 使用记录截图）
- `03-数据统计分析_测试文档` — Word测试文档（保留 Word 模板封面/修订记录/静态章节，按统计分析结果表拆分测试内容 + 解析工单自测报告内嵌截图 + 更新测试结论总数表）
- `04-数据统计分析_结果表及使用说明` — Excel说明材料（数据源表list + 表融合关系 + 结果表list + 结果表详情）

**材料类型由用户在指令中直接指定，不在台账清单中维护。**

## 使用方式

### 01-API接口开发 需求文档
```
使用 document-filler 技能，帮我生成服务目录为"N07-API接口开发"的"01-API接口开发_需求文档"文档材料，
台账清单的路径在 XXX，模版文件的路径在 XXX，输出的文件放在 XXX 路径下
```
台账需包含 `服务目录`、`需求单号`、`工单号`、`工单标题`、`程序数`、`工单描述`、`结果表清单` 和 `附件`。生成器按合并行聚合工单，从每个工单的内嵌 Word 自测报告“测试内容”章节提取接口输入/输出参数；接口作用描述会先概括工单标题和工单描述作为背景，再说明接口输入、输出和用途。输出参数为文件夹时，默认文件名为 `01-需求文档.docx`。附件或接口参数无法匹配时停止生成并报告具体工单或接口，不输出占位文档。
生成动态内容时以模板现有示例段落和表格为格式原型，保留直接字体、字号、缩进、行距、编号、项目符号、边框、底纹和单元格格式；工单标题和 API 接口标题沿用模板的 Word 自动编号并按出现顺序递增。

### 02-API接口开发 数据模型设计
```
使用 document-filler 技能，帮我生成服务目录为"N07-API接口开发"的"02-API接口开发_数据模型设计"文档材料，
台账清单的路径在 XXX，模版文件的路径在 XXX，输出的文件放在 XXX 路径下
```
台账需包含 `服务目录`、`需求单号`、`工单号`、`工单标题`、`程序数`、`工单描述`、`结果表清单` 和 `附件`。生成器复用 API 需求文档的台账聚合和自测报告解析逻辑，只替换模板 `接口出入参配置表` 到 `接口组件配置表` 之间的示例内容；大标题使用 `工单标题`，小标题使用接口中文名，入参/出参配置表来自自测报告中对应接口的输入/输出参数。输出参数为文件夹时，默认文件名为 `02- 数据模型设计（API）.docx`。
生成动态内容时优先复制模板中第 4.2 节的 `Heading 3`、`Heading 4`、`Heading 5` 和输入/输出表格原型，保留字体、字号、标题层级、表格列宽、边框、底纹和长表跨页表头；`请求头`、`请求体`、`userInfo`、`IdAuthData` 等参数分组名用模板正文段落格式保留。

### 03-API接口开发 接口开发代码
```
使用 document-filler 技能，帮我生成服务目录为"N07-API接口开发"的"03-API接口开发_接口开发代码"文档材料，
台账清单的路径在 XXX，模版文件的路径在 XXX，输出的文件放在 XXX 路径下
```
台账需包含 `服务目录`、`需求单号`、`工单号`、`工单标题`、`程序数`、`工单描述`、`结果表清单` 和 `附件`。生成器复用 API 需求文档的台账聚合和附件定位逻辑：`API接口开发列表` 表格按台账接口顺序生成，序号自增，接口代码使用接口英文名，接口名称使用接口中文名，责任委办固定填 `上海市大数据中心`；`接口模型明细` 按接口中文名生成标题，并从对应自测报告 `测试内容` 章节中匹配同名接口，依次插入 `共享接口命名规范性` 和 `共享任务开发代码检查` 两张截图。输出参数为文件夹时，默认文件名为 `03-接口开发代码.docx`。
生成动态内容时复制模板的接口列表表格、接口明细标题和图片段落原型，保留表格边框、底纹、单元格字体、标题层级、截图段落对齐和截图显示宽度。未提及的模板静态内容保持不变，并在生成后更新目录。

```
使用 document-filler 技能，帮我生成服务目录为"N07-API接口开发"的"04-API接口开发_接口测试报告"文档材料，
台账清单的路径在 XXX，模版文件的路径在 XXX，输出的文件放在 XXX 路径下
```
台账需包含 `服务目录`、`需求单号`、`工单号`、`工单标题`、`程序数`、`工单描述`、`结果表清单` 和 `附件`。生成器复用 API 需求文档的台账聚合和附件定位逻辑：`API测试清单` 表格按台账接口顺序生成，序号自增，接口代码使用接口英文名，接口名称使用接口中文名，责任委办固定填 `上海市大数据中心`；`API测试内容` 按接口生成标题 `{接口中文名} {接口英文名}`，将对应接口的输入参数、输出参数分别整理为一张表，并插入该接口 `测试结果` 小节下的截图。输出参数为文件夹时，默认文件名为 `04-接口测试报告（含《API接口列表》）.docx`。
旧版 `.doc` 模板会先临时转换为 `.docx`，最终输出为 `.docx`。生成动态内容时复制模板的测试清单表格、接口标题、输入/输出参数标题和参数表格原型；同一接口有多张测试结果截图时仅取第一张，截图前固定插入 `测试结果与预期结果一致，测试通过。`，截图说明和截图段落使用模板测试结果正文段落格式；未提及的模板静态内容保持不变，并在生成后更新目录。

```
使用 document-filler 技能，帮我生成服务目录为"N07-库表落地方式"的"02-库表落地方式_数据模型设计"文档材料，
```

### 必须参数

| 参数 | 说明 | 适用材料类型 |
|------|------|-------------|
| **服务目录** | 台账清单中"服务目录"列的值 | 全部 |
| **材料类型** | 模板文件名（不含 .docx） | 全部 |
| **台账清单路径** | 台账清单 Excel 文件完整路径 | 全部 |
| **模板文件路径** | Word 模板(.docx) 或 Excel 模板(.xlsx) 完整路径 | 全部 |
| **输出文件路径** | 生成的 Word 或 Excel 说明材料保存路径；用户给文件夹时按材料类型生成文件名 | 全部 |
| **数据目录数据路径** | 数据目录数据 Excel 文件路径；01-数据报表需求文档中为可选增强参数，用于生成更具体的数据需求和交付要求 | 01可选、02、04 |

## 执行流程

1. **读取台账清单**：打开 Excel，动态读取所有列
2. **筛选数据**：仅按 `服务目录` 列筛选匹配的行
3. **按材料类型分发**：根据 `material_type` 执行对应的填充逻辑
4. **提取附件和图片**：N07 API 接口开发类材料从 `附件` 列的自测报告提取接口参数表和接口代码检查截图；03 数据报表从台账清单 cellimages.xml 提取上线截图和使用记录截图；03 数据统计分析从 `附件` 列的工单自测报告提取测试截图；04 为表融合关系生成血缘流程图并内嵌
5. **打开模板**：用户指定的模板文件路径
6. **填充内容**：按填充规则生成各章节内容
7. **更新/校验输出**：Word 材料通过 PowerShell COM 调用 Word 自动更新目录；Excel 材料执行工作簿有效性与图片嵌入检查
