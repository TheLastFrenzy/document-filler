# 文档填充规则详细说明

## 台账清单列结构（动态读取，不硬编码列数）

Excel 第一行为列名，脚本动态读取所有列。筛选仅按 `服务目录` 列进行。

## 01-数据报表_需求文档

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：第3节标题(Heading 2) + 表格工单内容列
- `报表统计次数`：数值，用于统计
- `业务说明`：第3节业务说明内容
- `数据需求`：第3节数据需求内容
- `交付要求`：第3节交付要求内容

### 2.3 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据报表服务。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单内容 | 报表统计次数 |
|------|---------------|-------------|---------|-------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |
| 合计(合并前2列) | - | - | - | K |

### 第3节 需求内容填充规则

每条筛选记录生成以下结构：
```
Heading 2: {工单内容}
  Heading 3: 业务说明
    Body Text: {业务说明内容}  (首行缩进480twips)
    [如有报表清单: 描述行 "本次工作拟产出以下N份报表成果：" + 内嵌表格]
  Heading 3: 数据需求
    Body Text: {数据需求内容}  (首行缩进480twips)
  Heading 3: 交付要求
    Body Text: {交付要求内容}  (首行缩进480twips)
```

#### 内嵌Excel清单表格触发条件
在"业务说明"字段中检测模式：
- `本次工作拟产出以下N份报表成果` （带"以下"）
- `本次工作拟产出N份报表成果`    （不带"以下"）

匹配后：
1. 业务说明正文只保留标记之前的文字（去除清单列表）
2. 添加描述段落：`本次工作拟产出以下N份报表成果：`
3. 紧随其后插入 2 列表格（序号 + 名称），每行一个报表名称

解析规则：标记之后按行拆分，过滤空行和过短行，去掉前导数字编号。

## 01-数据统计分析_需求文档

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：第3节标题(Heading 2) + 表格工单内容列
- `报表统计次数`：数值，用于统计
- `业务描述`：第3节业务描述内容
- `统计分析结果表清单`：换行分隔的"中文名 英文表名"对
- `业务逻辑`：OLE Word附件，自动提取并总结为业务逻辑说明表格
- `数据统计分析执行周期`：数据加工周期描述
- `数据更新要求`：更新频率描述
- `数据量对后续运维的特殊要求`：运维特殊要求

### 2.3 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据统计分析。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列，列宽比例 1:2:2:4:1）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单标题 | 数据统计分析次数 |
|------|---------------|-------------|---------|----------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |

- 合计行合并前两列，最后一列显示报表统计次数之和
- 表格使用实线边框 + 固定布局 (tblLayout fixed)

### 3 需求内容填充规则

每个工单生成以下结构：

```
Heading 2: {工单内容}

  Heading 3: 业务描述
    Body Text: {业务描述}（首行缩进480）
    Body Text: 本次工作拟产出{N}个统计分析结果表。
    [统计分析结果表清单表格]

  Heading 3: 业务逻辑
    [业务逻辑说明表格：两列（步骤|说明）]

  Heading 3: 数据加工周期
    Body Text: 数据统计分析执行周期：{值}。
    Body Text: 数据更新要求：{值}。
    Body Text: 数据量对后续运维的特殊要求：{值}。
```

### 业务逻辑附件提取

- 通过VML shape定位"业务逻辑"列中的OLE Word附件
- 解析 `xl/drawings/vmlDrawing1.vml` 获取shape的row/col锚点
- 通过 `xl/worksheets/_rels/sheet1.xml.rels` 获取OLE对象路径
- 使用olefile库提取WordDocument流，UTF-16LE解码
- 自动总结为分步骤的业务逻辑说明表格

### 格式要求

- 表格：实线边框（single, sz=4），表头加粗+背景色 F1F1F1
- 需求来源表格列宽比例 1:2:2:4:1，固定布局
- 业务逻辑说明表格：标题行跨2列居中加粗，子标题行"步骤"、"说明"
- Body Text 段落：首行缩进 `firstLine=480`
- 未提及需填充的位置直接保留模板原文
- TOC：通过 PowerShell COM 调用 Word 自动更新

## 02-数据报表_设计文档

### 核心列（按列名匹配）

- `工单内容`：第3节每个小节的标题(Heading 2)
- `内容描述`：业务分析表格的内容描述
- `业务场景`：业务分析表格的业务场景
- `数据内容`：业务分析表格的数据内容
- `结果形式`：业务分析表格的结果形式
- `数据处理逻辑`：数据处理逻辑章节内容（文字或内嵌图片）
- `02-数据报表_设计文档-数据内容截图`：数据内容章节的内嵌图片
- `02-数据报表_设计文档-数据来源库表清单对应数据目录代码`：换行分隔的数据目录代码列表

### 额外参数

- `--catalog`：数据目录数据Excel路径（必填），包含两个Sheet：
  - "关联资源信息"：根据数据目录代码匹配资源名称和资源编码
  - "数据项"：根据数据目录代码匹配字段信息（数据项名称、英文名称、数据类型、字段描述）

### 第3节数据填充规则

每条筛选记录生成以下结构：
```
Heading 2: {工单内容}
  Heading 3: 业务分析
    表格(左右结构, 左侧标题加粗+背景色):
      | 内容描述 | {内容描述} |
      | 业务场景 | {业务场景} |
      | 数据内容 | {数据内容} |
      | 结果形式 | {结果形式} |
  Heading 3: 数据内容
    [内嵌截图图片]
  Heading 3: 数据来源库表清单
    引导文字: "为完成本次数据报表，需使用到以下数据："
    Heading 4: {资源编码} (每个数据目录代码一个)
      表格(字段信息):
        | 字段中文名称 | 字段英文名称 | 数据类型 | 备注 |
        [表头为"资源名称 资源编码"的合并行]
    [无匹配时: "未匹配到此目录的任何表结构信息，请手动补充。"]
  Heading 3: 数据处理逻辑
    Body Text: {数据处理逻辑} 或 内嵌图片
  Heading 3: 报表指标设计
    默认指标表格(7列, 10行数据)
```

### 表格格式

- 所有表格使用实线边框（single, sz=4）
- 业务分析表格：左右结构，左侧标题列背景色 F1F1F1 + 加粗
- 数据来源库表清单表格：首行合并4列显示"资源名称 资源编码"，第二行为表头行（加粗+背景色）
- 报表指标设计表格：表头加粗+背景色
- 字段描述为空时统一填充"No"

### 格式要求（所有材料类型通用）

- Body Text 段落：首行缩进 `firstLine=480` (约2个中文字符)
- 合计行：前两列合并为一个单元格 (gridSpan=2)
- TOC：通过 PowerShell COM 调用 Word Application 自动更新

## 03-数据报表_上线记录

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：各小节标题(Heading 1)，格式为"{工单内容}的上线记录"
- `报表统计次数`：数值，用于统计和表格
- `需求描述`：产出说明章节的需求描述内容
- `上线交付截图1`、`上线交付截图2`：上线交付截图章节的内嵌图片（DISPIMG 公式引用）
- `使用记录截图1`、`使用记录截图2`：使用记录章节的内嵌图片（DISPIMG 公式引用）

### 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据报表服务。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单内容 | 报表统计次数 |
|------|---------------|-------------|---------|-------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |
| 合计(合并前2列) | - | - | - | K |

表格使用实线边框（single, sz=4）。

### 各工单章节填充规则

每条筛选记录生成以下结构：
```
Heading 1: {工单内容}的上线记录
  Heading 2: 产出说明
    Body Text: 需求编号：{需求单号}    对应工单编号：{工单号}
    Body Text: 需求描述：{需求描述内容}  (首行缩进480twips)
    Normal: 统计报表：{报表统计次数}次。
  Heading 2: 上线交付截图
    Body Text: [内嵌上线交付截图1]
    Body Text: [内嵌上线交付截图2]
  Heading 2: 使用记录
    Body Text: 使用记录如下：
    Body Text: [内嵌使用记录截图1]
    Body Text: [内嵌使用记录截图2]
```

### 图片提取

- 图片从台账清单的 `xl/cellimages.xml` 中提取（WPS 格式）
- DISPIMG 公式中的 ID 与 cellimages.xml 中的 image name 匹配
- 图片路径通过 `xl/_rels/cellimages.xml.rels` 映射到 `xl/media/` 下的实际文件

### 格式要求

- 表格：实线边框（single, sz=4），表头加粗+背景色 F1F1F1
- Body Text 段落：首行缩进 `firstLine=480`
- TOC：通过 PowerShell COM 调用 Word 自动更新，锚定"需求来源"段落前
