# 文档填充规则详细说明

## 台账清单列结构（动态读取，不硬编码列数）

Excel 第一行为列名，脚本动态读取所有列。筛选仅按 `服务目录` 列进行。

## 01-数据报表_需求文档

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：第3节标题(Heading 2) + 表格工单内容列
- `报表统计次数`：数值，用于统计
- `业务说明`：第3节业务说明内容
- `数据需求`：第3节数据需求内容
- `交付要求`：第3节交付要求内容

### 2.3 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据报表服务。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单内容 | 报表统计次数 |
|------|---------------|-------------|---------|-------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |
| 合计(合并前2列) | - | - | - | K |

### 第3节 需求内容填充规则

每条筛选记录生成以下结构：
```
Heading 2: {工单内容}
  Heading 3: 业务说明
    Body Text: {业务说明内容}  (首行缩进480twips)
    [如有报表清单: 描述行 "本次工作拟产出以下N份报表成果：" + 内嵌表格]
  Heading 3: 数据需求
    Body Text: {数据需求内容}  (首行缩进480twips)
  Heading 3: 交付要求
    Body Text: {交付要求内容}  (首行缩进480twips)
```

#### 内嵌Excel清单表格触发条件
在"业务说明"字段中检测模式：
- `本次工作拟产出以下N份报表成果` （带"以下"）
- `本次工作拟产出N份报表成果`    （不带"以下"）

匹配后：
1. 业务说明正文只保留标记之前的文字（去除清单列表）
2. 添加描述段落：`本次工作拟产出以下N份报表成果：`
3. 紧随其后插入 2 列表格（序号 + 名称），每行一个报表名称

解析规则：标记之后按行拆分，过滤空行和过短行，去掉前导数字编号。

## 01-数据统计分析_需求文档

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：第3节标题(Heading 2) + 表格工单内容列
- `报表统计次数`：数值，用于统计
- `业务描述`：第3节业务描述内容
- `统计分析结果表清单`：结果表中文名 + 英文表名；同一工单可拆分为多行，工单信息合并单元格时自动向下继承并聚合
- `程序XML文本`：保留台账中的程序 XML 文本；需求文档的业务逻辑说明不直接读取此列
- `数据统计分析执行周期`：数据加工周期描述
- `数据更新要求`：更新频率描述
- `数据量对后续运维的特殊要求`：运维特殊要求

### 2.3 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据统计分析。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列，列宽比例 1:2:2:4:1）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单标题 | 数据统计分析次数 |
|------|---------------|-------------|---------|----------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |

- 合计行合并前两列，最后一列显示报表统计次数之和
- 表格使用实线边框 + 固定布局 (tblLayout fixed)

### 3 需求内容填充规则

每个工单生成以下结构：

```
Heading 2: {工单内容}

  Heading 3: 业务描述
    Body Text: {业务描述}（首行缩进480）
    Body Text: 本次工作拟产出{N}个统计分析结果表。
    [统计分析结果表清单表格]

  Heading 3: 业务逻辑
    [业务逻辑说明表格：两列（步骤|说明）]

  Heading 3: 数据加工周期
    Body Text: 数据统计分析执行周期：{值}。
    Body Text: 数据更新要求：{值}。
    Body Text: 数据量对后续运维的特殊要求：{值}。
```

### 业务逻辑说明生成

- 优先读取模板同目录的 `04-数据统计分析_结果表及使用说明.xlsx`。
- 在 `2、表融合关系` sheet 中按统计分析结果表英文名匹配 `文字描述`。
- 同一工单有多个结果表时，每个结果表汇总为业务逻辑说明表格中的一个步骤。
- 同一工单只有一个结果表时，将该结果表的 `文字描述` 按编号、换行、句号或分号拆分为多个步骤，避免全部集中在一个步骤里。
- 如果未找到 `04-数据统计分析_结果表及使用说明.xlsx` 或文字描述为空，保留旧版 OLE 附件提取作为兜底。

### 格式要求

- 表格：实线边框（single, sz=4），表头加粗+背景色 F1F1F1
- 需求来源表格列宽比例 1:2:2:4:1，固定布局
- 业务逻辑说明表格：标题行跨2列居中加粗，子标题行"步骤"、"说明"
- Body Text 段落：首行缩进 `firstLine=480`
- 未提及需填充的位置直接保留模板原文
- TOC：通过 PowerShell COM 调用 Word 自动更新

## 02-数据报表_设计文档

### 核心列（按列名匹配）

- `工单内容`：第3节每个小节的标题(Heading 2)
- `内容描述`：业务分析表格的内容描述
- `业务场景`：业务分析表格的业务场景
- `数据内容`：业务分析表格的数据内容
- `结果形式`：业务分析表格的结果形式
- `数据处理逻辑`：数据处理逻辑章节内容（文字或内嵌图片）
- `02-数据报表_设计文档-数据内容截图`：数据内容章节的内嵌图片
- `02-数据报表_设计文档-数据来源库表清单对应数据目录代码`：换行分隔的数据目录代码列表

### 额外参数

- `--catalog`：数据目录数据Excel路径（必填），包含两个Sheet：
  - "关联资源信息"：根据数据目录代码匹配资源名称和资源编码
  - "数据项"：根据数据目录代码匹配字段信息（数据项名称、英文名称、数据类型、字段描述）

### 第3节数据填充规则

每条筛选记录生成以下结构：
```
Heading 2: {工单内容}
  Heading 3: 业务分析
    表格(左右结构, 左侧标题加粗+背景色):
      | 内容描述 | {内容描述} |
      | 业务场景 | {业务场景} |
      | 数据内容 | {数据内容} |
      | 结果形式 | {结果形式} |
  Heading 3: 数据内容
    [内嵌截图图片]
  Heading 3: 数据来源库表清单
    引导文字: "为完成本次数据报表，需使用到以下数据："
    Heading 4: {资源编码} (每个数据目录代码一个)
      表格(字段信息):
        | 字段中文名称 | 字段英文名称 | 数据类型 | 备注 |
        [表头为"资源名称 资源编码"的合并行]
    [无匹配时: "未匹配到此目录的任何表结构信息，请手动补充。"]
  Heading 3: 数据处理逻辑
    Body Text: {数据处理逻辑} 或 内嵌图片
  Heading 3: 报表指标设计
    默认指标表格(7列, 10行数据)
```

### 表格格式

- 所有表格使用实线边框（single, sz=4）
- 业务分析表格：左右结构，左侧标题列背景色 F1F1F1 + 加粗
- 数据来源库表清单表格：首行合并4列显示"资源名称 资源编码"，第二行为表头行（加粗+背景色）
- 报表指标设计表格：表头加粗+背景色
- 字段描述为空时统一填充"No"

### 格式要求（所有材料类型通用）

- Body Text 段落：首行缩进 `firstLine=480` (约2个中文字符)
- 合计行：前两列合并为一个单元格 (gridSpan=2)
- TOC：通过 PowerShell COM 调用 Word Application 自动更新

## 02-数据统计分析_设计文档

此材料输出为 Word `.docx`，不额外生成 PDF。入口使用 `scripts/fill_document.py`，材料类型为 `02-数据统计分析_设计文档`。

### 核心数据来源

- `--excel`：台账清单。按 `服务目录` 筛选 `N08-数据统计分析`，同一工单拆成多行、工单信息使用合并单元格时自动继承并聚合。
- `--template`：`02-数据统计分析_设计文档.docx` 模板。
- `--catalog`：数据目录数据 Excel，读取 `关联资源信息` sheet，用 `资源编码` 匹配源表/结果表。
- 模板同目录 `04-数据统计分析_结果表及使用说明.xlsx`：读取 `1、数据源表list`、`2、表融合关系`、`4、数据统计分析结果表详情`。

### 需求来源章节

表格按每个统计分析结果表生成一行：

| 序号 | 需求单编号 | 工单编号 | 工单名称 | 程序名称 |
|------|------------|----------|----------|----------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 统计分析结果表英文名 |

描述文字中的需求单数、工单数、程序总数分别按去重需求单、去重工单、统计分析结果表数量计算。

### 数据统计分析设计章节

对需求来源中的每个程序生成一组详细内容。若一个工单涉及多个统计分析结果表，必须为每个结果表都生成一组内容。

#### 标题

- Heading 2：统计分析结果表中文名

#### 涉及到的实体表

源表来自 `04-数据统计分析_结果表及使用说明.xlsx` 的 `1、数据源表list`：

- 通过统计分析结果表英文名匹配 `数据融合加工表`
- 源表表名取 `资源信息（表名）`
- 数据目录编码取 `资源编目（非必填）`，为空时用数据目录数据的 `数据目录代码`
- 数据目录中文名称取 `资源名称`
- 表类型固定为 `源表`
- 数据更新周期通过源表英文名匹配数据目录数据 `关联资源信息.资源编码` 后取 `业务数据更新周期`

每个结果表最后追加一行目标表：

- 数据目录编码：通过结果表英文名匹配数据目录数据 `关联资源信息.资源编码` 后取 `数据目录代码`
- 数据目录中文名称：取 `资源名称`
- 表名：统计分析结果表英文名
- 表类型固定为 `目标表`
- 数据更新周期：取 `业务数据更新周期`

#### 数据统计分析设计

- Heading 4：`统计分析结果表中文名 + 空格 + 统计分析结果表英文名`
- 结果表结构来自 `4、数据统计分析结果表详情`

字段映射：

| 文档列 | 来源 |
|--------|------|
| 字段中文名 | 字段中文名 |
| 字段英文名 | 字段英文名 |
| 字段类型 | 字段类型 |
| 是否为空 | 优先取 `是否为空`，兼容取 `不可为空` |
| 主键/外键 | 优先取 `主键/外键`，兼容取 `唯一` |
| 字段说明 | 字段注释，空值填 `No` |

#### 数据处理流程图

从 `2、表融合关系` 中按统计分析结果表英文名匹配 `数据处理流程图` 单元格的 `DISPIMG` 图片，并插入 Word。

#### 数据加工逻辑

从 `2、表融合关系` 中按统计分析结果表英文名匹配 `文字描述`，按编号、换行、句号或分号拆分为“数据统计分析加工逻辑说明”步骤表。

## 03-数据统计分析_测试文档

此材料输出为 PDF。入口使用 `scripts/fill_document.py`，材料类型为 `03-数据统计分析_测试文档`。

### 核心数据来源

- `--excel`：台账清单。按 `服务目录` 筛选 `N08-数据统计分析`，同一工单拆成多行时自动继承合并单元格。
- `--template`：`03-数据统计分析_测试文档.pdf` 模板。最终 PDF 直接复用模板封面、修订记录和 1-4 章静态内容，避免重绘非填充页面。
- 台账列 `统计分析结果表清单`：格式为 `统计分析结果表中文名 + 空格 + 统计分析结果表英文名`。
- 台账列 `03-数据统计分析_测试文档_工单自测报告附件`：每个工单内嵌一个 Word 自测报告附件，附件覆盖该工单所有统计分析结果表。

### 自测报告解析

从台账 Excel 的 OLE `package` 流提取内嵌 Word 自测报告，按所在行或合并区域关联工单。自测报告中读取 `4.检查和测试内容` 章节，按程序英文名匹配以下七类截图：

1. 程序中英文名称规范性
2. 表命名规范性
3. 表字段名规范性
4. 表及表字段注释规范性
5. 程序逻辑检查
6. 程序代码注释规范性
7. 程序运行测试

若台账英文名和自测报告英文名只有后缀差异或相似度高，允许近似匹配并在程序标题下写明匹配说明。

### PDF 生成规则

- 模板页保留：直接保留模板第 1 页封面、第 2 页修订记录，以及模板中 1-4 章静态内容页。
- 目录页重建：根据实际程序清单重建目录，包含 5.x 程序级条目和 `6.测试结论`，并写入 PDF 内部可点击跳转链接，不能让目录链接都回到第一页。
- `5.测试内容`：按台账中每个统计分析结果表生成一节，标题使用结果表中文名。
- 每个程序下生成七个小节，截图按自测报告中出现顺序插入；同一小节多张图时依次插入。
- 如果某工单没有找到自测报告附件，仍保留该工单所有程序和七个小节，内容标注“截图待补充”。
- `6.测试结论`：正文使用模板原版结论描述；下方表格只更新总数：

| 测试内容 | 测试项 | 通过项 | 通过率 |
|----------|--------|--------|--------|
| 数据测试 | 程序总数 | 程序总数 | 100% |
| 程序测试 | 程序总数 | 程序总数 | 100% |

## 04-数据统计分析_结果表及使用说明

此材料输出为 Excel `.xlsx` 工作簿，不是 Word 文档。入口仍使用 `scripts/fill_document.py`，材料类型为 `04-数据统计分析_结果表及使用说明`，并且必须提供 `--catalog` 数据目录数据路径。

### 核心列（按列名匹配）

- `服务目录`：筛选值通常为 `N08-数据统计分析`
- `程序XML文本`：draw.io/mxGraphModel 形式的程序流转 XML，内含 SQL、表达式、节点标签等
- `统计分析结果表清单`：结果表中文名 + 英文表名

### 额外参数

- `--catalog`：数据目录数据Excel路径（必填），用于补齐源表/结果表资源信息和字段信息。
- `--template`：Excel模板 `.xlsx`，应包含以下 sheet：
  - `说明`
  - `1、数据源表list`
  - `2、表融合关系`
  - `3、数据统计分析结果表list`
  - `4、数据统计分析结果表详情`

### 工作簿生成规则

1. 按 `服务目录` 从台账清单筛选记录。
2. 对每条记录解析 `程序XML文本`：
   - 从 SQL 中识别结果表、源表、关联表和 CTE。
   - 排除结果表自身、`DUAL`、`ALL_TABLES` 等非业务源。
   - 从 create/insert/select 片段提取结果表字段；如 XML 无字段，则回退到数据目录数据字段。
3. 结合数据目录数据补齐：
   - 源表/结果表的资源名称、资源编码、资源类型、单位名称、数据目录代码。
   - 数据项名称、英文名称、数据类型、字段描述。

### Sheet 1：`1、数据源表list`

按解析出的源表去重后生成源表清单：

| 资源名称 | 数据提供方 | 资源类型 | 数据源表的资源信息（表名） |
|----------|------------|----------|----------------------------|

### Sheet 2：`2、表融合关系`

每个结果表固定生成 3 行：

1. 标题行：`2.{序号} {结果表英文名}`，合并 A:C。
2. `文字描述` 行：C列写业务化分点总结。
3. `数据处理流程图` 行：C列内嵌流程图图片。

#### 文字描述要求

- 不能直接复制 XML 文本、SQL 片段、节点标题或注释。
- 必须分析 XML/SQL 后，用业务可读语言分点总结结果表的数据梳理逻辑。
- 推荐结构：
  1. 数据来源准备：说明读取哪些源表作为基础数据。
  2. 数据筛选清洗：总结有效记录、分区批次、状态、区域、服务范围等筛选口径。
  3. 融合加工计算：总结 join、union、group by、count、case、环比/同比/占比等加工逻辑。
  4. 结果字段整理：列出主要输出字段和运行信息。
  5. 结果输出：说明写入哪个结果表，供数据统计分析应用使用。

#### 数据处理流程图要求

- 主要反馈源表到结果表的血缘关系，不展示 XML 程序步骤细节。
- 样式参考模板：黑色背景、浅橙色节点、深灰色箭头。
- 结构为：`开始` -> 一个或多个源表节点 -> `统计融合为 {结果表中文名/英文名}` -> `推送至应用开发部` -> `结束`。
- 多源表时使用并行源表节点汇聚到结果表；源表过多时只展示主要源表并用“其他N张源表”归并。
- 源表节点优先显示资源名称 + 英文表名；英文长表名按下划线分组换行，避免单词被拆得不可读。

### Sheet 3：`3、数据统计分析结果表list`

每个结果表生成一行：

| 资源名称 | 数据提供方 | 资源类型 | 数据统计分析表的资源信息（表名） |
|----------|------------|----------|----------------------------------|

### Sheet 4：`4、数据统计分析结果表详情`

每个结果表生成一个字段详情区块：

| 字段中文名 | 字段英文名 | 字段类型 | 默认 | 不可为空 | 唯一 | 字段注释 |
|------------|------------|----------|------|----------|------|----------|

字段优先来自 XML 中的结果表建表/插入语句，XML 不完整时使用数据目录数据补齐。字段描述为空时填 `No`。

### 验证要求

- 生成的 `.xlsx` 必须能被 Excel 打开。
- `2、表融合关系` 的结果表段数应等于筛选出的结果表数量。
- `2、表融合关系` 的内嵌图片数量应等于结果表数量。
- 扫描 `文字描述`，不得出现 `mxGraphModel`、`<mxCell`、`create table`、`insert into`、`truncate table`、`sql语句` 等原始 XML/SQL 痕迹。

## 03-数据报表_上线记录

### 核心列（按列名匹配）

- `需求单号`：对应需求单编号
- `工单号`：对应工单编号
- `工单内容`：各小节标题(Heading 1)，格式为"{工单内容}的上线记录"
- `报表统计次数`：数值，用于统计和表格
- `需求描述`：产出说明章节的需求描述内容
- `上线交付截图1`、`上线交付截图2`：上线交付截图章节的内嵌图片（DISPIMG 公式引用）
- `使用记录截图1`、`使用记录截图2`：使用记录章节的内嵌图片（DISPIMG 公式引用）

### 需求来源填充规则

#### 描述文字
格式：`服务周期内，共有{N}张需求单，{M}张工单涉及{K}次数据报表服务。具体需求单、工单和产出如下表：`
- N = 去重后需求单号数量
- M = 去重后工单号数量
- K = 报表统计次数之和

#### 表格（5列）
| 序号 | 对应需求单编号 | 对应工单编号 | 工单内容 | 报表统计次数 |
|------|---------------|-------------|---------|-------------|
| 自增 | 需求单号 | 工单号 | 工单内容 | 报表统计次数 |
| 合计(合并前2列) | - | - | - | K |

表格使用实线边框（single, sz=4）。

### 各工单章节填充规则

每条筛选记录生成以下结构：
```
Heading 1: {工单内容}的上线记录
  Heading 2: 产出说明
    Body Text: 需求编号：{需求单号}    对应工单编号：{工单号}
    Body Text: 需求描述：{需求描述内容}  (首行缩进480twips)
    Normal: 统计报表：{报表统计次数}次。
  Heading 2: 上线交付截图
    Body Text: [内嵌上线交付截图1]
    Body Text: [内嵌上线交付截图2]
  Heading 2: 使用记录
    Body Text: 使用记录如下：
    Body Text: [内嵌使用记录截图1]
    Body Text: [内嵌使用记录截图2]
```

### 图片提取

- 图片从台账清单的 `xl/cellimages.xml` 中提取（WPS 格式）
- DISPIMG 公式中的 ID 与 cellimages.xml 中的 image name 匹配
- 图片路径通过 `xl/_rels/cellimages.xml.rels` 映射到 `xl/media/` 下的实际文件

### 格式要求

- 表格：实线边框（single, sz=4），表头加粗+背景色 F1F1F1
- Body Text 段落：首行缩进 `firstLine=480`
- TOC：通过 PowerShell COM 调用 Word 自动更新，锚定"需求来源"段落前
